## 1. RTK（Rust Token Killer）インストール

Claude Code のトークン消費を削減するため RTK v0.38.0 を導入した。

### 手順

```powershell
# GitHub Releases から Windows バイナリを取得・展開
$url  = "https://github.com/rtk-ai/rtk/releases/latest/download/rtk-x86_64-pc-windows-msvc.zip"
$dest = "$env:LOCALAPPDATA\rtk"
Invoke-WebRequest -Uri $url -OutFile "$env:TEMP\rtk.zip"
Expand-Archive "$env:TEMP\rtk.zip" -DestinationPath $dest -Force

# ユーザー PATH に追加
$p = [System.Environment]::GetEnvironmentVariable("PATH","User")
[System.Environment]::SetEnvironmentVariable("PATH","$p;$dest","User")
```

Claude Code フック設定（`~/.claude/settings.json`）:

```json
"hooks": {
  "PreToolUse": [
    {"matcher": "Bash",
     "hooks": [{"type": "command", "command": "rtk hook claude"}]}
  ]
}
```

- プロジェクト CLAUDE.md に RTK コマンドリストを追記（`rtk init`）
- グローバルフック設定は `settings.json` に手動追記

### 結果

`rtk gain` でトークン節約統計を確認できる状態になった。  
git / grep / ls 等のコマンドが自動的に RTK 経由でフィルタリングされる。

## 2. Table5 AE 頻度表 — Grade 5 列追加

### 問題

ユーザーが AE 頻度表マクロ（`ae_row`）を G5 対応に手修正済みだったが、出力に Grade 5 列が表示されなかった。

### 原因

`proc report` の `column` 文に `c6` が追加されていなかった。`define c6` は存在するが `column` 文に列名が**ない**と非表示になる SAS の仕様。

```sas
/* 修正前 */
column c1 ("...Grade" c2 c3 c4 c5);

/* 修正後 */
column c1 ("...Grade" c2 c3 c4 c5 c6);
```

### 対応

PowerShell の文字列置換で `column` 文に `c6` を追加。  
Shift-JIS で書き戻し（`[System.IO.File]::WriteAllText($path, $content, CP932)`）。

**コミット**: `4b7587f`

## 3. Table5 Section 5（感染症起因菌一覧）— 列幅調整試行→元に戻す

### 試みた変更

SAS の `proc report` の `cellwidth` 単位指定・ヘッダ定義に問題があったと推測されるが、デバッグよりも元の動作優先をユーザーが選択。

### 対応

元の `proc print width=min` に戻した。
