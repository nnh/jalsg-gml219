- 変更前：`cga_phase = "bl"` / `cga_phase = "c1"`
- 変更後：`cga_phase = "_bl"` / `cga_phase = "_c1"`
- 背景：SAS 実行後のログで CGA 変数 28 件の WARNING を検出。診断プログラムで調査したところ、この環境（SAS 9.4 TS1M7 DBCS3170）では `PROC TRANSPOSE` の `id` に複数変数を指定した際、アンダースコアなしで値が連結される（例：`CGA1A` + `bl` → `CGA1Abl`）ことが判明。`cga_phase` の値に先頭アンダースコアを付けることで期待する変数名（`CGA1A_bl`）を生成するよう対処
