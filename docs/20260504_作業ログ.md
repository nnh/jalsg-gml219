| | コード |
|---|---|

### 修正3: `cga_phase` の値にアンダースコア接頭辞を追加

| | コード |
|---|---|
| 修正前 | `if qsspid = "baseline_cga" then cga_phase = "bl";` |
| 修正後 | `if qsspid = "baseline_cga" then cga_phase = "_bl";` |

原因: SAS 9.4 TS1M7 DBCS3170 では `PROC TRANSPOSE` の `id` で値が連結される際、アンダースコアなしだと `CGA1Abl` になる。`_bl` にすることで `CGA1A_bl` を生成。

### 修正4: 打ち切り日をカットオフ日 → 個人の最終追跡日に変更（EFS/OS/RFS 共通）

コミット: `7fd9902`

| | コード |
|---|---|
| 修正前 | `efsdt = &cutdt.`（2025-12-05 固定） |
| 修正後 | `efsdt = dsstdt2`（DS DSSPID="withdrawal" の DSSTDTC） |

検出契機: 症例 GML219-0006 の打ち切り日が固定値になっており個人の最終追跡日ではなかった。

DS ドメインの構造:
- `DSSPID="discontinuation"` + `EPOCH="TREATMENT"` → 治療終了日（打ち切り日ではない）
- `DSSPID="withdrawal"` + `EPOCH="FOLLOW-UP"` → 最終追跡日 / 死亡日（こちらを使う）

### 修正5: EFS 治療不応イベント日を DS 中止日 → RS 評価日に変更

コミット: `ee068f6`

| | コード |
|---|---|
| 修正前 | `if dsterm = "LACK OF EFFICACY" then edt_fail = dsstdt;` |
| 修正後 | eval2あり → `edt_fail = rsdt_ev2`、eval1のみ → `edt_fail = rsdt_ev1` |

原因: DS 中止日は RS 評価日の数日後になることが多く、イベント日は効果判定日（RS ドメインの RSDTC）が正しい。

同コミットで crfl 判定も修正: `where rsspid in ("evaluation1","evaluation2")` を追加（地固め以降の評価を除外）。

残存問題: FTMCC + CR=N の 4 例（0002, 0046, 0120, 0128）が処理漏れ。

### 修正6: EFS 治療不応を RS 評価結果から直接判定（根本修正）

コミット: `99dfbfc`

```sas
if crfl ne "Y" then do;
  if rsdt_ev2 ne . then edt_fail = rsdt_ev2;
  else if rsdt_ev1 ne . then edt_fail = rsdt_ev1;
end;
```

| DS 中止理由 | crfl | eval2 | 件数 | 処理 |
|---|---|---|---|---|
| LACK OF EFFICACY | N | あり | 15例 | eval2 RSDTC をイベント日 |
| FAILURE TO MEET CONTINUATION CRITERIA | N | なし（eval1のみ） | 4例 | eval1 RSDTC をイベント日 |
| FAILURE TO MEET CONTINUATION CRITERIA | Y | — | 11例 | EFSイベントなし（打ち切り） |

### 未解決: label 文の `%` エラー

