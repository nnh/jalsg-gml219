# JALSG-GML219 プログラム改修記録

作成日: 2026-05-05  
対象リポジトリ: https://github.com/nnh/jalsg-gml219  

---

## 改修の背景

2025年12月に作成した解析プログラム群（`program/202512 pgm v1/`）を2026年5月に再実行したところ、以下の問題が顕在化した。

1. **Table1** — `proc tabulate` の class 文に現 gml219 に存在しない旧変数名（`bl_chromXXXyn`、`bl_cdXXXyn` 等）が並びエラー
2. **Table5（旧 AE テーブル）** — `length c1 . c2-c5 .;` になっており（`$200.` が消えた）集計不能
3. **Fig0** — `working_dir` マクロのバッククォートバグでパス解決失敗
4. **Table6/7/8 未作成**
5. **RFS 起算日の不一致** — PRT v2.4 §8.1.1 は「RFS 起算 = LFS 到達日」と規定しているが実装は `crdt`（CR 到達日）を使用
6. **SAP 5章の対象項目が大幅に不足**（CGA、ELN2017、CCI 等が Table1 に未含）

## 1. CSVtoSASDS 大規模修正（2026-05-04、commit: 5a1950f）

### 1A. 追加インポートドメイン

MB（起因菌）、PR（後治療）ドメインを `%sasds_sdtm` 呼び出しに追加。

### 1B. 新規セクション §14a〜§14i

| セクション | 内容 | 主要変数 |
|---|---|---|
| §14a | 後治療・放射線（PR） | `pttx_radio_fl`, `pttx_radio_dt` |
| §14b | 造血細胞移植（PR） | `pttx_hsct_fl`, `pttx_hsct_dt`, `pttx_graft`, `pttx_donor` |
| §14c | 移植前情報（FA） | `pttx_beftrans`（病期）, `pttx_hla`（HLA 一致度） |
| §14d | 移植情報（SC） | `pttx_trump`, `pttx_jmdp` |
| §14e | 移植前処置（CM） | `pttx_cond`, `pttx_cond_dt` |
| §14f | 試験治療完了フラグ（DS） | `compfl`（Y = 完了/COMPLETED） |
| §14g | LFS・治療不応フラグ | `lfsfl`, `lfsdt`（= crdt 近似）, `tffl`, `tftype` |
| §14h | 起因菌（MB） | `mb_i1`〜`mb_c3`（コース別菌名文字列） |
| §14i | SAE long データ | `gml219_sae.sas7bdat`（PT × コース） |

### 1C. RFS 起算日修正（PRT v2.4 §8.1.1 準拠）

```sas
/* 修正前: rfs_d = rfsdt - crdt + 1  */
/* 修正後: */
rfs_d = rfsdt - lfsdt + 1;   /* lfsdt = crdt（保守的近似） */
```

LFS と CR の違い:
- **CR**: 骨髄芽球 < 5% ＋ 好中球 > 1000/μL ＋ 血小板 > 100000/μL ＋ 輸血非依存 ＋ 髄外病変なし（5条件）
- **LFS**: 骨髄芽球 < 5% ＋ 髄外病変なし（2条件）
- RS ドメインに CRi/LFS 専用値がないため `lfsdt = crdt` で保守的近似

### 1D. データセット規模変化

| | 修正前 | 修正後 |
|---|---|---|
| gml219.sas7bdat | 128 obs × 369 vars | 128 obs × 428 vars |
| gml219_sae.sas7bdat | 未作成 | 24 obs × 5 vars |

## 2. 解析プログラム修正（2026-05-04〜05）

### Table1（患者背景）— 全面リライト（commit: 5a1950f）

**問題**: 旧変数名（bl_chromdel5qyn 等）を使用しており変数が存在しなかった。

主な変数名対応:

| 旧変数名 | 新変数名 |
|---|---|
| `dxmhterm` | `dxwhoterm`, `fabclass` |
| `bl_chromdel5qyn` | `del5q` |
| `bl_chrominv16yn` | `inv16` |
| `bl_chromt821yn` | `t821` |
| `bl_cd33yn` | `CD33` |
| `bl_geneflt3yn` | `flt3itd` |
| `bl_genenpm1yn` | `npm1` |
| `bl_ptinr` | `bl_inr` |
| `bl_bblast`, `bl_pblast` | `bl_blastle`, `bl_myblale` |

追加項目（SAP 5.2.1 準拠）: CGA（cga*a_bl/cga*b_bl）、CCI（cci_bl、cci_bl_* 疾患別）、ELN2017、血液異常既往（bl_bldabn）、8週以内感染（bl_infect8w）

### Table2（治療概要）— 変数名修正（commit: e9f7733）

**問題**: 変数名が現 gml219 と不一致。

| 旧変数名 | 新変数名 |
|---|---|
| `i1_AraCdose` | `ind1aradose` |
| `i1_DNRdose` | `ind1dnrdose` |
| `i1_AraCdays` | 計算式 `ind1enddt - ind1stdt + 1` で代替 |
| `c1_MITdose` | `c1mitdose` |
| `c3_ACRdose` | `c3acrdose` |

### Table3（CR率・LFS率・治療不応率）— LFS/治療不応率を新規追加（commit: 5a1950f）

**修正前**: CR率のみ出力  
**修正後**: CR率・LFS率（lfsfl 使用）・治療不応率（tffl/tftype 使用）・CR/LFS 到達日数を追加

### Table4（WT-1 mRNA）— バグ修正 2件（commit: e9f7733）

1. **working_dir バッククォートバグ**: `%scan(...,-1,\)` → `%scan(...,-1,'\')`
2. **length 型宣言バグ**: `length timepoint ord 8.` → `length timepoint $40. ord 8. wt1 8.`

### Table5（有害事象）— 全面リライト（commit: 5a1950f）、Grade5 追加（commit: 4b7587f）

**修正前の問題**:
- `length c1 . c2-c5 .;`（PowerShell `@"..."@` が `$200.`/`$30.` を消した）
- `%ae_table(i1, fasfl="Y", ...)` → `=` がマクロエラー
- `ls=120` が短すぎて c1 幅エラー

**修正後の構成**:
- Section 1: AE 頻度（Any/G1-2/G3/G4/G5）コース別
- Section 2: 血液所見最低値（WBC/Neut/Hb/PLT）LB ドメインから
- Section 3: 好中球 < 1000/μL 持続日数（FA ドメイン DURATION）
- Section 4: 深在性真菌症（FA ドメイン DPFUNINF、EORTC/MSG 基準）
- Section 5: 感染症起因菌一覧（MB ドメイン由来 mb_i1〜mb_c3）

**Grade 5 追加時の注意点**（2026-05-05）:
- `ae_row` マクロ内の `length`・`retain`・カウントロジックに c6/g5 を追加するだけでなく
- **`proc report` の `column` 文にも `c6` を追加すること**（define 文があっても column 文がなければ非表示）

### Table6（重篤有害事象）— 新規作成（commit: 08a53db）

gml219_sae.sas7bdat（AESER="Y"）とコース終了日から、投与終了 28 日以内の SAE を PT × コース別に集計。

### Table7（死亡詳細）— 新規作成（commit: 08a53db）

prcdth の文字列マッチで死亡分類（腫瘍死・移植関連死・非腫瘍死・不明）を派生。

### Table8（後治療）— 新規作成（commit: 08a53db）

CSVtoSASDS §14a〜§14e で追加した `pttx_*` 変数群を使用。  
移植有無・病期・移植片種類・ドナー・HLA・前処置・JMDP・TRUMP 番号を集計。
